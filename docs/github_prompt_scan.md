# GitHub prompt scan

Дата: 2026-05-15.

## Что проверено

Быстрый поиск по публичным GitHub/Gist материалам дал несколько типов источников:

- общие фреймворки prompt engineering;
- humanizer/AI-writing-tells чеклисты;
- простые промпты про perplexity и burstiness;
- инструменты, которые используют API или локальные CLI.

Для CheckAI нельзя использовать API, локальные LLM и ручную смысловую правку, поэтому из таких репозиториев нельзя брать workflow целиком. Можно брать только идеи для первичного промпта в веб-чате.

## Полезные идеи

1. Структурировать промпт как задачу с контекстом, целью, правилами, стилем, форматом ответа и критериями оценки.
2. Давать explicit negative constraints: чего не должно быть в тексте.
3. Просить внутреннюю проверку, но запрещать выводить ее наружу.
4. Отдельно контролировать ритм текста: длина предложений, длина абзацев, повтор переходов.
5. Отдельно контролировать предметную конкретику, потому что один только "human style" часто делает текст разговорным, но слабым по качеству.
6. Не полагаться на общую фразу "add perplexity and burstiness"; ее лучше переводить в конкретные требования: разная длина абзацев, меньше повторов, меньше симметрии, больше терминов по теме.

## Что не используем

- Humanizer-сервисы.
- CLI/скрипты для переписывания текста.
- API-based evaluation.
- Локальные модели.
- Ручное редактирование сгенерированного документа.
